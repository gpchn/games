项目概述

项目名称：智学问答助手——基于 AI 的交互式知识学习平台

目标：为学校师生打造一款智能问答学习工具，系统可根据所选主题（党史、校史、学科知识等）随机出题，用户作答后由 AI 实时判断对错并生成个性化解析，帮助巩固记忆、拓展知识。

核心价值
“出题—答题—解析”闭环学习，融入学校特色内容，成为信息化教育创新载体。
展示亮点
AI教育应用 + 可扩展性设计，体现技术对学习方式的重构。

模块设计
4.1 多主题题库管理模块

数据结构：每道题包含 id, topic (党史/校史/数学等), question, options (数组), answer, explanation (备选解析), difficulty。按主题分别存储为JSON文件，便于独立更新与扩展。

示例：data/party_history.json，data/school_history.json，未来可加入学科题库。教师可直接编辑JSON文件添加新题。
4.2 随机出题模块

后端 GET /api/question/random?topic=party_history 根据主题随机抽取题目；若不指定主题则从所有主题中随机选取。算法采用 random.choice，保证每次独立随机。

4.3 答题与判题模块

后端 POST /api/answer/submit 接收用户答案，比对正确答案后调用AI生成个性化解析，并返回正确性标记与解析文本。保底机制：AI失败时返回题库预置 explanation。

4.4 前端界面模块

单页应用风格，包含主题选择区、题目展示区、选项单选、提交按钮及结果区。支持“下一题”刷新，响应式设计融入学校文化元素（校徽、主题色）。

交互流程：加载随机题 → 用户作答 → 提交 → 展示AI解析 → 下一题（保持主题）
数据流程 & API设计

}

POST /api/answer/submit

}

响应示例：

  "is_correct": true,
  "explanation": "正确！中共一大于1921年在上海召开，后转移到嘉兴南湖。"
}

前端调用API后渲染结果，全程无状态，支持高并发。

AI集成详细设计
模型选择

推荐使用中文优化模型：Qwen/Qwen2.5-7B-Instruct 或 THUDM/glm-4-9b-chat，在事实性问答和解析生成上表现优异，免费额度满足中小规模使用。备选：deepseek-ai/DeepSeek-R1-Distill-Qwen-7B。

API调用封装（OpenAI兼容）
import os, requests
def call_llm(prompt: str) -> str:
    api_key = os.environ.get("LLM_API_KEY")
    base_url = os.environ.get("LLM_BASE_URL")
    model = os.environ.get("LLM_MODEL", "Qwen/Qwen2.5-7B-Instruct")
    headers = {"Authorization": f"Bearer {api_key}", "Content-Type": "application/json"}
    payload = {
        "model": model,
        "messages": [{"role": "user", "content": prompt}],
        "temperature": 0.7,
        "max_tokens": 300
    }
    resp = requests.post(f"{base_url}/chat/completions", json=payload, headers=headers, timeout=5)
    return resp.json()["choices"][0]["message"]["content"] if resp.ok else ""
Prompt 设计
你是一位知识渊博的老师，擅长用简洁准确的语言解释知识点。
主题：{topic}
题目：{question}
选项：{options}
正确答案：{correct_answer}
用户答案：{user_answer}
请输出：
- 判断结果：正确/错误
- 解析：（不超过100字，说明为什么对/错，并补充相关背景知识）
容错与成本控制：设置max_tokens≤300，相同题目+相同答案短期缓存；若AI调用失败则降级使用题库预置解析。

Vercel 部署步骤

  1. 将代码推送至GitHub仓库
  2. 在Vercel中导入项目，设置环境变量 LLM_API_KEY, LLM_BASE_URL, LLM_MODEL
  3. 点击Deploy，自动构建并生成可访问域名
  4. 支持自定义域名绑定，展示给全校师生使用
完全免费部署，无服务器维护成本
安全性考虑

输入校验（question_id合法性）、CORS同域无需额外配置、简单IP限流防止恶意请求、AI输出敏感词过滤，确保内容合规。
